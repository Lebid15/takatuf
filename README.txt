==============================================
نظام الصيدليات الزراعية
Agricultural Pharmacy Management System
==============================================

طريقة التشغيل
-------------
- افتح ملف index.html مباشرة من المتصفح (Chrome / Edge / Firefox).
- لا يحتاج النظام إلى خادم أو تنصيب أي شيء.
- النظام يعمل بالكامل من المتصفح.

الأقسام داخل النظام
-------------------
- لوحة التحكم: ملخص سريع للأرقام والحركات الأخيرة.
- الزبائن: إضافة وعرض وحذف الزبائن مع عرض الرصيد المتبقي.
- مستودعي: إدارة المنتجات (مبيدات، أسمدة، بذور...) مع متابعة الكميات والتنبيهات.
- الحركات: تسجيل مبيعات الزبائن بتعبئة تلقائية لاسم المنتج والسعر.
- الدفعات: تسجيل الدفعات الجزئية والكاملة من الزبائن.
- كشف الحساب: عرض مفصل لكل زبون (مدين / دائن / المتبقي) مع طباعة احترافية.
- الإعدادات: تعديل بيانات الصيدلية، الشعار، تغيير كلمة المرور.

تخزين البيانات
--------------
- جميع البيانات محفوظة محليًا داخل متصفحك (localStorage).
- لا يتم رفع أي بيانات إلى أي خادم خارجي.
- كل حساب يملك بياناته الخاصة المعزولة عن بقية الحسابات.
- تنظيف بيانات المتصفح سيمسح بيانات النظام؛ احرص على أخذ نسخة احتياطية بشكل دوري.

النسخ الاحتياطي والاستيراد
--------------------------
- لأخذ نسخة احتياطية: اذهب إلى "كشف الحساب" → اضغط "تصدير نسخة احتياطية".
  سيتم تنزيل ملف JSON باسم agri-pharmacy-backup-YYYY-MM-DD.json يحتوي بيانات الحساب الحالي.
- لاستيراد نسخة: اضغط "استيراد نسخة" واختر الملف.
  سيُطلب منك تأكيد قبل استبدال البيانات الحالية.

ميزات إضافية
-----------
- دعم كامل للعربية (RTL).
- تصميم متجاوب يعمل على الشاشات المختلفة.
- زر "إظهار/إخفاء" لكل حقل كلمة مرور.
- زر طباعة منفصل لكل قسم مع قالب احترافي يشمل الشعار وبيانات الصيدلية.
- ميزة "نسيت كلمة المرور" مع كود تحقق تجريبي محلي.

ملاحظة للمطور
-------------
يمكن لاحقًا ربط النظام بـ Backend (مثل Django) وقاعدة بيانات حقيقية،
وإرسال بريد إلكتروني فعلي لاستعادة كلمة المرور.
بُنية الكود في script.js تفصل بين دوال العرض (render*) ودوال البيانات
(load/save) ودوال الحساب لتسهيل الترحيل لاحقًا.
