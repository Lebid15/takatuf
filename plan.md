الثالث عشر: النسخ الاحتياطي والاستيراد
======================================

أضف زر:
تصدير نسخة احتياطية

عند الضغط:

* يتم تحميل ملف JSON يحتوي بيانات الحساب الحالي فقط.

اسم الملف:
agri-pharmacy-backup-YYYY-MM-DD.json

أضف زر:
استيراد نسخة

عند اختيار ملف JSON:

* تحقق من صحة الملف.
* اسأل المستخدم:
  سيتم استبدال بيانات الحساب الحالي بالنسخة المستوردة، هل تريد المتابعة؟
* عند الموافقة يتم استيراد البيانات للحساب الحالي فقط.

==================================================
الرابع عشر: حذف البيانات
========================

زر حذف كل البيانات:

* يظهر داخل النظام بعد تسجيل الدخول.
* يحذف بيانات الحساب الحالي فقط:

  * الزبائن.
  * الحركات.
  * الدفعات.
  * المستودع.
  * الإعدادات المتعلقة بالصيدلية.
* لا يحذف كل الحسابات.
* لا يحذف حسابات أخرى.
* اطلب تأكيدًا واضحًا قبل الحذف.

==================================================
الخامس عشر: إدارة البيانات داخليًا
==================================

استخدم localStorage.

اقترح بنية بيانات مثل:

{
accounts: [
{
id: "account-id",
pharmacyName: "",
ownerName: "",
email: "",
password: "",
phone: "",
address: "",
logoData: "",
customers: [],
inventory: [],
sales: [],
payments: [],
createdAt: ""
}
],
activeAccountId: ""
}

أو أي بنية منظمة مشابهة.

مهم جدًا:

* كل البيانات يجب أن تكون مفصولة حسب الحساب.
* لا تستخدم بيانات عامة مشتركة بين كل الحسابات.
* عند تسجيل الدخول، يتم تحميل بيانات الحساب الحالي فقط.
* عند تسجيل الخروج، يتم إغلاق الجلسة.

استخدم sessionStorage لحفظ حالة تسجيل الدخول الحالية، مثل:
activeAccountId

التاسع عشر: شروط القبول النهائية
================================

قبل تسليم المشروع، اختبر الآتي:

1. فتح index.html مباشرة من المتصفح.
2. تظهر صفحة دخول عامة باسم:
   نظام الصيدليات الزراعية
3. لا يظهر اسم صيدلية محددة في صفحة الدخول.
4. يمكن إنشاء حساب جديد.
5. عند إنشاء الحساب يدخل المستخدم للنظام مباشرة.
6. يظهر اسم الصيدلية داخل النظام بعد الدخول.
7. يمكن تسجيل الخروج.
8. يمكن تسجيل الدخول بالبريد وكلمة المرور.
9. زر العين يعمل في كل حقول كلمة المرور.
10. نسيت كلمة المرور تفتح صفحة استعادة مستقلة.
11. يمكن تعيين كلمة مرور جديدة باستخدام كود تحقق تجريبي محلي.
12. يمكن إضافة زبون.
13. يمكن إضافة منتج في مستودعي.
14. المنتج يظهر في قائمة المنتجات داخل إضافة حركة.
15. عند اختيار منتج يتم تعبئة الاسم والنوع والسعر تلقائيًا.
16. يتم حساب إجمالي الحركة تلقائيًا.
17. يتم احتساب الكمية المباعة من المستودع.
18. يتم احتساب الكمية المتوفرة.
19. تظهر حالة المنتج منخفض عند الوصول لحد التنبيه.
20. يمكن إضافة دفعة جزئية.
21. كشف الحساب يحسب المدين والدائن والمتبقي.
22. كل زر طباعة يطبع قسمه فقط.
23. قالب الطباعة يظهر فيه اللوغو وبيانات الصيدلية.
24. تصدير JSON يعمل.
25. استيراد JSON يعمل.
26. بيانات كل حساب منفصلة عن الحساب الآخر.
27. لا توجد أخطاء في Console.
28. التصميم واضح ومريح ويعمل على الشاشات المختلفة.
